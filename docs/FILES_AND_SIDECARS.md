# Files & Sidecars

MIVF uses sidecar files placed next to your `.mivf` video file for additional features.

## Supported Sidecar Files

Place any of these in the same folder as `yourvideo.mivf`:

### Subtitles

| File | Purpose |
| :--- | :--- |
| `yourvideo.srt` | Main subtitle track |
| `yourvideo.1.srt`, `yourvideo.2.srt`, … | Additional subtitle tracks |

Cycle through tracks during playback with **Y**. Subtitle delay and on-screen position can be adjusted in Settings.

### Chapters

| File | Purpose |
| :--- | :--- |
| `yourvideo.chapters` | Chapter markers |

Format: one entry per line.

```
# comment lines start with #
90 Opening
05:30 Chapter Name
1:12:45 Credits
```

- Plain seconds (`90`)
- `M:SS` or `H:MM:SS`
- Optional label after the timestamp

Navigate chapters during playback with **R + ←/→**.

### Cover Poster

| File | Purpose |
| :--- | :--- |
| `yourvideo.cover` | Browser preview poster image |

A raw RGB565 image file, exactly 88×50 pixels (the browser preview size). If not present, the player auto-decodes the first video frame as a thumbnail.

### Synopsis

| File | Purpose |
| :--- | :--- |
| `yourvideo.nfo` | Short description shown in the browser preview panel |

Plain text, displayed in the browser preview area.

### Seek Index

| File | Purpose |
| :--- | :--- |
| `yourvideo.idx` | Seek index sidecar for fast seeking |

Auto-generated by the encoder. The encoder also embeds a seek index footer directly in the `.mivf` file. See [SEEK_INDEX.md](SEEK_INDEX.md).

## App Data Layout

Settings, bookmarks, and other persistent data live under:

```
sdmc:/3ds/mivf_player_3ds/appdata/
├── settings.ini          Player settings
├── favorites.ini         Favorite video paths
├── recents.ini           Recently opened videos
├── bookmarks/            Per-video resume positions
├── logs/                 Debug logs
├── cache/                Cached data
└── benchmarks/           Benchmark data
```

Legacy root-level files (`sdmc:/mivf_settings.ini`, `sdmc:/mivf_favorites`, `sdmc:/mivf_recents`, `sdmc:/mivf_bookmarks/`) are still read for migration but new data is always written to the appdata folder.
