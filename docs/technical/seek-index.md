# Seek Index

MIVF supports fast seeking through seek indexes. Without an index, the player must scan through the file to find the right position, which is slow for large files.

## How It Works

The encoder generates a seek index that maps frame numbers to byte offsets in the `.mivf` file. The player uses this to jump directly to the right position.

## Index Formats

### Sidecar Index (`.idx`)

A separate file placed next to your `.mivf`:

```
yourvideo.mivf
yourvideo.idx
```

Generated by the encoder by default. To disable:

```bash
python encode_mivf.py input.mp4 output.mivf --no-seek-index
```

### Embedded Index (MIDX Footer)

A seek index footer appended directly to the end of the `.mivf` file. The player reads this metadata without needing a separate sidecar file.

Generated by default. To disable:

```bash
python encode_mivf.py input.mp4 output.mivf --no-embedded-index
```

The player checks for the embedded index first, then falls back to the sidecar `.idx`.

## Player Behavior

### With Index

- Opening a file: reads the index (embedded or sidecar), stores it in memory.
- Seeking: O(1) lookup, near-instant jump to the correct byte offset.
- First seek after open may still need a brief decoder warm-up.

### Without Index

- The player attempts a binary search through the file.
- For very large files on slow SD cards, this can cause a noticeable delay.
- The player caps the synchronous scan time to avoid freezing.

### Large Uncached Files

For very large files (>~256 MB) with no index, the player skips the expensive synchronous scan entirely. Seeking will be approximate (based on file position ratio). You can improve this by generating an index during encoding.

## Manual Index Generation

If you have an existing `.mivf` without an index:

```bash
python encode_mivf.py input.mp4 output.mivf --m2y2
```

This re-encodes with index generation. There is currently no standalone index-only tool.

## Performance

- Index files are small (typically <100 KB for a feature-length video).
- Reading the index at open time is fast — a single `fread` call.
- Embedded index is read from the file tail, same I/O cost as a sidecar.
